# CAJAL Figure Report — The Verified Brand Data Contract
_Density: recommend 4 figures, Mixed._

Four figurable concepts. First, the five epistemic-status labels (raw data, verified data, generated artifact, approval record, report) are a taxonomy that underpins the whole contract. Second, the broken-chain visualization — a claim traced back through intermediate artifacts to sources, with a labeled break — is the chapter's signature image and a process/provenance schematic. Third, the recipe structure (inputs → steps → outputs → gate → log) is a process flowchart that recurs across the book and is fully specified here. Fourth, the "looks done vs is defensible" two-column contrast opens the chapter and is a clean comparison panel. The agentic supervision triangle also appears here, but it is the same figure already specified for Chapter 2; rather than duplicate it, this report defers to the Chapter 2 supervision-frame figure and notes the reuse.

## Figure 1 — The Five Epistemic-Status Labels
Priority: Critical · Type: hierarchy/taxonomy

BLOCK 1 — ILLUSTRAE PASTE BLOCK:
Create a flat vector taxonomy for a single-column textbook page showing the five label types in the verified brand data contract. Arrange five parallel boxes in a vertical stack, ordered from raw to finished: raw data, verified data, generated artifact, approval record, and report. Each box carries a reserved label slot and a reserved area beside it for a short definition of what may and may not be cited from that label. Use color to mark epistemic status rather than quality: give raw data a neutral tone, verified data and approval record a supportive confirmed tone, the generated artifact a cautionary tone signaling it is not evidence by default, and the report a distinct dominant tone as the terminal product. ## Figure 2 — The Broken Provenance Chain
Priority: Critical · Type: process flowchart

BLOCK 1 — ILLUSTRAE PASTE BLOCK:
Create a flat vector provenance-chain diagram for a single-column textbook page that traces a single strategic claim backward to its sources. Place the claim at the top as one node. Draw lines descending from the claim through intermediate artifact nodes — a report, a template, a recipe run, a model summarization — down to a row of source-document nodes at the bottom. Along one of the descending lines, insert a clearly marked break in the line, a visible gap or severed connection, to show where the chain fails; reserve a label slot beside this break for the note that the source was not checked for date or methodology. Make one of the lower nodes a generated artifact, distinctly colored to mark it as not-evidence-by-default. The point is that the break does not invalidate the whole chain; it localizes where human review must focus, so keep the intact links visually solid and only the one break interrupted. Use single-headed downward arrows for the trace direction, thin strokes, flat fills, white background, and no baked-in words.

BLOCK 2 — FULL SCOPE:
[S - SPECIFICATION] Single-column 89mm, ≥300 DPI, vector, Okabe-Ito, white background.
[C - CONTENT] Top claim node; intermediate nodes (report, template, recipe run, model summarization); bottom source-document nodes. One marked break on a descending link [inferred placement] with reserved note slot. One node flagged as generated-artifact / not-evidence.
[O - ORGANIZATION] Vertical top-down trace; single-headed ↓ arrows; one interrupted link showing the break; reserved label slots at claim, break, and generated-artifact node.
[P - PRESENTATION] Flat vector, Okabe-Ito hex (claim #0072B2, intact links Black #000000, break #D55E00, generated artifact #D55E00, sources #56B4E9), 1pt strokes, white background, no baked text, no style suggestions.

## Figure 3 — The Recipe Structure

Make the Gate node visually distinct and the strongest element in the diagram, because the gate is the professional moment where the accountable human crosses from prepared ground to decision.
