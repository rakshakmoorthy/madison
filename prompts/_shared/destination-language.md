# destination-language.md (shared)

Replace **engine** language (how it works) with **destination** language (what the
reader/customer experiences). Describe the outcome, not the machinery. Used by any
suite that writes for a non-technical audience.

## Engine → Destination

| Engine term (remove) | Destination term (use) |
|---|---|
| AI pipeline / workflow | content system / the steps that run |
| n8n workflow automation | automated system |
| multi-agent orchestration | multiple specialists working in sequence |
| LLM | AI writing system |
| prompt engineering | training the system on your voice |
| vector embeddings | smart matching |
| knowledge base | story / reference library |
| API integration | connects your existing tools |
| Jungian archetype detection | understands brand personality |
| POC / proof of concept | here is what it already does |
| onboarding | getting started |
| TAM / SAM / SOM | the full scope of the problem |
| ARR | annual recurring revenue (spell out once, then use the number) |
| deck / slide | (remove — present as a document, not a presentation artifact) |

## Rule
Describe the **destination** (what the customer experiences), never the **engine**
(how it works). Structure as Before → After → the Bridge. End on the "Aha":
"Instead of [old painful way], [customer] can now [new simple outcome]."

## Forbidden phrases (rewrite before output)
- "Strong social media presence" → name the platform, the measure, the comparison.
- "Very engaged audience" → give the ratio / the substance.
- "Good brand consistency" → name one inconsistency, or confirm uniformity with evidence.
- "Leverage / synergy / seamless / cutting-edge / innovative" → delete.
- Hedges: "potentially / could possibly / might be able to" → delete.
- Self-congratulation: "We are thrilled to present / This innovative solution" → delete.
- Meta-commentary: "As you can see from this slide / Moving on to" → delete.
